# 🔑 KeyLog (Розумна Ключниця на ESP32)

![Platform](https://img.shields.io/badge/platform-ESP32-blue.svg)
![Framework](https://img.shields.io/badge/framework-Arduino%20IDE-orange.svg)
![UI](https://img.shields.io/badge/UI-Modern_Dark-black.svg)
![Status](https://img.shields.io/badge/status-Active-success.svg)

**KeyLog** — це IoT-проєкт розумної ключниці, створеної на базі мікроконтролера ESP32. Система відстежує наявність ключів у реальному часі за допомогою портів RJ-45, має стильний веб-дашборд та автоматично надсилає сповіщення у Telegram при зміні статусу ключів.

> 💡 **Походження ідеї:** Сама концепція використання корпусу старого мережевого комутатора (свіча) та конекторів RJ-45 у якості брелоків для ключів була знайдена на просторах інтернету. Цей проєкт є повноцінною авторською програмною та апаратною реалізацією цієї ідеї.

---

## ✨ Основні можливості

* **Telegram-сповіщення:** Миттєві повідомлення про те, який ключ взяли (🔴 DOWN) або повернули (🟢 UP).
* **Web-дашборд:** Вбудований веб-сервер із сучасним темним інтерфейсом (Modern Dark UI) для перегляду статусів та історії подій.
* **Captive Portal:** Початкове налаштування Wi-Fi без жорстко закодованих (hardcoded) паролів через зручний інтерфейс точки доступу.
* **Веб-конфігуратор:** Зміна імен ключів, логінів/паролів та Telegram-токенів прямо через браузер. Збереження в незалежну пам'ять (LittleFS).
* **NTP Синхронізація:** Точний час в історії подій завдяки синхронізації з серверами точного часу.
* **Апаратне скидання:** Можливість скинути всі налаштування затисненням кнопки `BOOT` на платі.

---

## 🛠 Апаратна частина (Hardware)

* **Мікроконтролер:** ESP-WROOM-32 (DevKit v1, 30-pin)
* **Корпус:** Будь-який старий мережевий свіч (наприклад, TP-Link на 8-16 портів).
* **Брелоки:** Звичайні конектори RJ-45 із замкнутими контактами 1 та 2.

### Схема підключення
Для стабільної роботи без хибних спрацьовувань використовуються піни з внутрішньою підтяжкою (INPUT_PULLUP), які не впливають на процес завантаження ESP32:

| RJ-45 Порт | Контакт RJ-45 | Пін ESP32 |
| :--- | :--- | :--- |
| **Усі порти** | Контакт 1 | `GND` (Спільна земля) |
| **eth1** | Контакт 2 | `GPIO 25` (D25) |
| **eth2** | Контакт 2 | `GPIO 26` (D26) |
| **eth3** | Контакт 2 | `GPIO 27` (D27) |
| **eth4** | Контакт 2 | `GPIO 32` (D32) |

## 💻 Програмне забезпечення (Software)

Для компіляції проєкту в **Arduino IDE** необхідні наступні бібліотеки:
1.  `WiFiManager` (від tzapu) — версія з підтримкою ESP32.
2.  `UniversalTelegramBot` (від Brian Lough).
3.  `ArduinoJson` — **ВАЖЛИВО: використовуйте версію 6.x** (наприклад, 6.21.4). Синтаксис 7-ї версії не підтримується цим кодом.
4.  Вбудовані бібліотеки ESP32: `WiFi`, `WebServer`, `LittleFS`.

## 🚀 Швидкий старт

1.  **Прошивка:** Зберіть схему та завантажте скетч `KeyLog.ino` в ESP32 через Arduino IDE.
2.  **Підключення до Wi-Fi:** При першому запуску ESP32 створить відкриту точку доступу **`KeyLog_Setup`**. Підключіться до неї з телефону або ПК.
3.  **Налаштування мережі:** У вікні, що з'явиться (Captive Portal), оберіть вашу домашню Wi-Fi мережу та введіть від неї пароль.
4.  **Вхід в Адмінку:** Знайдіть IP-адресу пристрою у вашій локальній мережі (через налаштування роутера) і перейдіть за нею у браузері.
    * *Логін за замовчуванням:* `admin`
    * *Пароль за замовчуванням:* `admin`
5.  **Конфігурація:** На сторінці веб-дашборду вкажіть назви ваших ключів, токен вашого Telegram-бота (від `@BotFather`) та ваш Chat ID. Натисніть "Зберегти".

---

## ⚠️ Скидання налаштувань
Якщо ви змінили роутер або забули пароль від веб-інтерфейсу, натисніть і утримуйте кнопку **`BOOT`** на платі ESP32 протягом 3-х секунд. Пристрій відформатує пам'ять, зітре налаштування Wi-Fi та перезавантажиться у режим точки доступу `KeyLog_Setup`.
